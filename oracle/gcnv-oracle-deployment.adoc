= Use Google Cloud NetApp Volumes to deploy high availability Oracle Database 26ai
:hardbreaks:
:nofooter:
:icons: font
:linkattrs:
:imagesdir: ../media/

[.lead]
Use Google Cloud NetApp Volumes iSCSI block storage to deploy high availability Oracle Database 26ai with Oracle Data Guard on Google Compute Engine. This workflow guides you through deployment options, infrastructure provisioning, Oracle software installation, and high availability configuration, leveraging GCNV's storage-layer replication for accelerated standby initialization.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png[One]link:gcnv-oracle-solution-architecture.html[Review the solution architecture]
[role="quick-margin-para"]
Understand the Oracle Database 26ai HA architecture, including reference data paths, platform roles, and storage topology.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png[Two]link:gcnv-oracle-deployment-options.html[Review deployment options and requirements]
[role="quick-margin-para"]
Compare deployment tiers from dev/test to production HA and confirm infrastructure, access, and software prerequisites.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png[Three]link:gcnv-oracle-access-and-apis.html[Verify access and APIs]
[role="quick-margin-para"]
Confirm required access, API enablement, networking prerequisites, and time synchronization.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png[Four]link:gcnv-oracle-provision-compute.html[Provision Google Compute Engine instances]
[role="quick-margin-para"]
Create VMs for primary database, standby database, and Fast-Start Failover Observer.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png[Five]link:gcnv-oracle-provision-storage.html[Provision GCNV iSCSI storage]
[role="quick-margin-para"]
Create GCNV storage pools and iSCSI volumes, configure multipath, and set up ASM disk groups.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png[Six]link:gcnv-oracle-install-software.html[Install Oracle Grid Infrastructure and Oracle Database 26ai]
[role="quick-margin-para"]
Install Oracle Grid Infrastructure and Oracle Database 26ai software on both database hosts.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-7.png[Seven]link:gcnv-oracle-primary-database.html[Create the primary database]
[role="quick-margin-para"]
Create the primary Oracle database with ASM storage and configure Oracle Restart.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-8.png[Eight]link:gcnv-oracle-standby-database.html[Create the standby database]
[role="quick-margin-para"]
Initialize the standby database using GCNV replication, snapshots, or clones, then register with Oracle Restart. All HA tiers complete this step.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-9.png[Nine]link:gcnv-oracle-standby-finalize-dg.html[Finalize standby database for Data Guard]
[role="quick-margin-para"]
Create standby redo logs, enable flashback, activate redo shipping, and verify Data Guard state. *Tier-specific:* Required only for Prod HA (Data Guard + FSFO) tier.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-10.png[Ten]link:gcnv-oracle-data-guard.html[Configure Data Guard Broker and Fast-Start Failover]
[role="quick-margin-para"]
Enable the broker, create the Data Guard configuration, enable FSFO, and configure the Observer. *Tier-specific:* Required only for Prod HA (Data Guard + FSFO) tier.
